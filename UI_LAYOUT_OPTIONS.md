## Implementation Notes

1. **Responsive Design**: On smaller screens, we can:
   - Stack columns vertically
   - Use a card layout instead of table
   - Add horizontal scroll with sticky first column

2. **Accessibility**: 
   - Add `aria-label` for screen readers
   - Ensure color contrast meets WCAG standards
   - Add tooltips explaining what each count means

3. **Performance**:
   - Table should render quickly with new column
   - No additional API calls needed (data already fetched)
